Core Grid System

Core Grid is a dynamic mobile first Sass/CSS grid. The grid is designed to have default break points for the majority of content layouts and a secondary semantic grid for fined tuned control over layouts.

Primary Default Grid

The first primary default is optimized for single column layout. All columns start at a width of 100%. Use the specified class name to trigger columns at the corresponding container width.

The class structure is the following (.column-width-breakpoint)

Example ".col-6-md"

  • ".col" defines that you want a column.
  • "-6" defines how many columns you want the column to span.
  • "-md" defines the default breakpoint you would like your column width to be triggered at.

Dynamic Layouts

Core Grid allows dynamic changing of column sizes. Example below we have four containers. By default they are 100% in width. At our "-sm" breakpoint they will span the width of 6 columns. The second class "-lg" defines our second change that will occur at the large breakpoint. This will trigger the columns to span 3 columns wide.

<section> 
    <div class="col-6-sm col-3-lg"> </div> 
    <div class="col-6-sm col-3-lg"> </div> 
    <div class="col-6-sm col-3-lg"> </div>
    <div class="col-6-sm col-3-lg"> </div>
</section>

Secondary Control Grid

A secondary grid is for the fined tuned control over your breakpoints. All .col start at 100% mobile first. To enable cols at certain width use media queries. Secondary grid follows the naming convention col-1-c (-c for custom denotation). 

<section class="page-class"> 
    <div class="col-6-c"></div> 
    <div class="col-6-c"></div> 
</section>

@media all and (min-width: 33em)
    .page-class {
        .col-6-c {
            width:49%;
        }
    }
}

Utilities

By leveraging Sass we can have a true mobile first responsive grid and still support older IE versions. Using special Sass mixins we can create a seperate old IE style sheet for a fixed grid for our media queries to support ie8 and below. To read more on this please read Jake Archibalds great blog post here.

Core Grid contains multiple helper classes to show and hide content to the corresponding grid breakpoints.
